## 一、项目背景与目标

### 1.1 项目背景

#### 1.1.1 产品现状
- **产品名称**：智慧笔记（原"投资笔记"）
- **核心功能**：用户记录思考笔记，AI模拟导师回复
- **当前导师**：6位（查理·芒格、巴菲特、段永平、张小龙、乔布斯、马斯克）
- **技术栈**：微信小程序原生框架 + 微信云开发 + DeepSeek API

#### 1.1.2 转型动因
| 动因 | 说明 |
|------|------|
| **合规风险** | "投资"主题存在金融合规风险 |
| **用户价值** | 导师的通用思想价值远高于投资标签 |
| **市场机会** | 个人成长与思考工具市场空白 |
| **差异化竞争** | 多导师思想碰撞是独特创新点 |

### 1.2 项目目标

#### 1.2.1 总体目标
打造一款**个人成长与思考笔记工具**，通过多领域导师的智慧碰撞，帮助用户深入思考、获得启发、实现成长。

#### 1.2.2 核心设计原则
| 原则 | 说明 |
|------|------|
| **界面极简** | 遵循微信/Apple设计规范，无冗余元素，交互自然直观 |
| **内容详尽** | 文档覆盖全流程，细节完备，逻辑清晰 |
| **渐进迭代** | 小步快跑，每阶段交付单一功能，测试通过后进入下一阶段 |
| **兼容优先** | 所有升级完全兼容现有版本，不影响用户使用体验 |

#### 1.2.3 阶段性目标

| 阶段 | 时间 | 目标 |
|------|------|------|
| **第一阶段** | 1-2个月 | 完成去投资化，扩展导师库至16-20位 |
| **第二阶段** | 2-3个月 | 实现多导师圆桌会议、讨论稿生成与导出、完善质量控制体系 |
| **第三阶段** | 3-6个月 | 实现思想孵化器、思想博弈场等创新功能 |

### 2.2 核心功能详细需求

#### 2.2.1 导师系统

**功能描述**：管理四大领域的导师库，支持导师选择、筛选、推荐。

**导师矩阵**：

| 领域 | 导师数量 | 核心导师 |
|------|---------|---------|
| **投资领域** | 4位 | 查理·芒格、巴菲特、段永平、格雷厄姆 |
| **创业领域** | 5位 | 张小龙、乔布斯、马斯克、贝佐斯、彼得·蒂尔 |
| **心理学领域** | 5位 | 卡尔·荣格、弗洛伊德、弗洛姆、阿德勒、马斯洛 |
| **哲学领域** | 7位 | 老子、孔子、苏格拉底、柏拉图、亚里士多德、尼采、维特根斯坦 |

**功能要点**：
- 导师卡片展示：头像、姓名、头衔、核心标签、领域
- 领域筛选：按四大领域筛选导师
- 智能推荐：根据用户问题内容推荐合适导师
- 历史偏好：记录用户偏好导师

#### 2.2.2 单导师对话（现有功能优化）

**功能描述**：用户选择一位导师，进行单轮或多轮对话。

**交互流程**：
```
用户选择导师 → 输入问题 → AI生成回复 → 18小时延迟 → 用户查看回复
```

**优化要点**：
- 提示词优化：去除投资相关表述
- 回复质量：增加质量控制机制
- 上下文记忆：支持多轮对话上下文

#### 2.2.3 多导师圆桌会议（新增核心功能）

**功能描述**：用户选择3-5位导师，形成圆桌会议讨论。

**交互流程**：
```
用户输入问题 → 选择3-5位导师 → 系统智能排序 → 
依次生成回复（每位导师基于前文） → 形成讨论稿 → 用户可追问
```

**技术要点**：
- 导师排序算法：按领域顺序、思维深度、角色互补
- 上下文传递：每位导师回复包含前文摘要
- 讨论稿生成：结构化输出，包含总结

#### 2.2.4 思想孵化器（创新功能）

**功能描述**：用户输入一个想法，多位导师帮助"孵化"这个想法。

**交互流程**：
```
用户输入想法 → 系统分解维度 → 不同导师从不同角度审视 → 
多轮迭代优化 → 形成完整思想体系
```

**技术要点**：
- 想法解构引擎：将想法分解为多个维度
- 多维评估系统：每个维度由合适导师评估
- 迭代优化算法：多轮迭代优化想法

#### 2.2.5 看懂「公司」的结构形状（新增特色功能）
**参考来源**：[公司结构分析框架 - 开源版本](https://gist.github.com/bill20232033cc/970d8a1c1ac607a136af98bbcfb4cd3b)
**版本**：v1.0 - 2026-03-17

**功能描述**：根据用户输入的公司相关笔记内容，自动生成结构化的公司结构分析，帮助用户深度理解商业组织底层运作逻辑。

**设计理念**：万物皆为"结构"，而结构是在压力下由向心力与离心力动态平衡形成的"涡漩体"。通过「三才结构」框架，拆解公司在不同发展阶段的能量运作逻辑。

**核心分析框架**：
```
公司结构分析体系
├── 结构动力学分析（三才结构）
│   ├── 天（外部压力/Environment）：时代基因、环境制约、熵增威胁
│   ├── 人（向心力/Core）：核心凝聚力量、组织内核、竞争优势
│   └── 地（离心力/Boundary）：业务边界、产品矩阵、生态版图
├── 结构形态判定（Vortex Status）
│   ├── 物理/几何隐喻定义（如：高速旋转的钻头、加厚堡垒、粘滞蛛网）
│   └── 形态生存逻辑解释
└── 涡旋结构快照（ASCII Snapshot）
    ├── 外部压力可视化
    ├── 涡漩主体结构
    ├── 力量方向标示
    └── 关键节点标注
```

**交互流程**：
```
用户输入公司相关笔记/新闻/财报 → 系统识别公司及发展阶段 → 
调用适配领域导师 → 生成结构化分析报告 → 展示ASCII结构快照 → 
用户可进一步提问深度分析
```

**适配导师组合**：
- 投资领域：查理·芒格、巴菲特、格雷厄姆（价值判断）
- 创业领域：贝佐斯、彼得·蒂尔、马斯克（商业逻辑）
- 其他领域：根据行业特性动态推荐

**分析报告结构**：
```
## 📊 「{公司名称}」结构分析报告

### 🔍 分析阶段：{公司发展阶段}

### 🧬 结构动力学分析
**天（外部压力）**：{时代背景、市场环境、竞争压力}
**人（向心力）**：{核心团队、技术优势、企业文化、护城河}

### 🎯 结构形态判定
**形态隐喻**：{具体物理/几何隐喻，如："高速旋转的锋利钻头"}
**生存逻辑**：{解释为什么在当时的时空环境下，必须是这个形状才能生存}

### 🎨 涡旋结构快照 (ASCII)
```
{ASCII艺术结构图，展示压力、核心、边界、力量方向}
```

### 💡 关键洞察
- {洞察点1：结构优势}
- {洞察点2：潜在风险}
- {洞察点3：未来演化方向}

### 🚀 行动建议
- {建议1：战略层面}
- {建议2：战术层面}

```

**技术实现要点**：
1. NLP识别公司名称、行业、发展阶段
2. 自动匹配对应领域的导师组合
3. 结构化信息提取与分析
4. ASCII艺术图动态生成
5. 支持多维度对比分析
6. 分析报告支持导出分享

#### 2.2.6 看懂「产品」的结构形状（新增特色功能）
**参考来源**：[公司结构分析框架 - 开源版本](https://gist.github.com/bill20232033cc/970d8a1c1ac607a136af98bbcfb4cd3b)（产品定制版）
**版本**：v1.0 - 2026-03-17

**功能描述**：根据用户输入的产品相关笔记内容（需求文档、竞品分析、用户反馈等），自动生成结构化的产品结构分析，帮助用户深度理解产品的底层逻辑与演化路径。

**设计理念**：产品是"需求-能力-边界"动态平衡的结构系统，在市场压力、团队能力和用户需求的共同作用下形成特定的结构形态。通过改良版「三才结构」框架，拆解产品在不同生命周期的结构特征与演化逻辑。

**核心分析框架**：
```
产品结构分析体系
├── 结构动力学分析（改良三才结构）
│   ├── 天（外部压力/Market）：市场环境、竞争格局、用户需求变化、技术趋势
│   ├── 人（核心能力/Core）：产品定位、核心价值、技术壁垒、团队能力
│   └── 地（边界拓展/Boundary）：功能矩阵、用户覆盖、生态边界、商业化路径
├── 结构形态判定（Product Status）
│   ├── 产品形态隐喻（如：尖刀型产品、平台型生态、臃肿型产品、极简型工具）
│   └── 形态适配逻辑解释
└── 产品结构快照（ASCII Snapshot）
    ├── 市场压力可视化
    ├── 产品核心能力层
    ├── 功能边界扩展方向
    └── 关键矛盾点标注
```

**交互流程**：
```
用户输入产品相关笔记/需求/竞品分析 → 系统识别产品类型、生命周期阶段 → 
调用适配领域导师 → 生成结构化分析报告 → 展示ASCII结构快照 → 
用户可进一步提问优化方向
```

**适配导师组合**：
- 产品领域：张小龙、乔布斯、贝佐斯（产品哲学）
- 创业领域：马斯克、彼得·蒂尔（创新逻辑）
- 心理学领域：阿德勒、荣格（用户认知）
- 其他领域：根据产品行业特性动态推荐

**分析报告结构**：
```
## 📱 「{产品名称}」结构分析报告

### 🔍 分析阶段：{产品生命周期阶段}

### 🧬 结构动力学分析
**天（外部压力）**：{市场环境、竞品格局、用户需求变化、技术趋势}
**人（核心能力）**：{产品定位、核心价值主张、技术壁垒、团队能力优势}

### 🎯 结构形态判定
**形态隐喻**：{产品形态隐喻，如："精准刺穿市场的尖刀型产品"、"生态繁茂的平台型产品"}
**适配逻辑**：{解释为什么在当前市场环境下，这个产品结构能够生存/失败}

### 🎨 产品结构快照 (ASCII)
```
{ASCII艺术结构图，展示市场压力、核心能力层、功能边界、演化方向}
```

### 💡 关键洞察
- {洞察点1：产品核心优势}
- {洞察点2：潜在结构缺陷}
- {洞察点3：未来演化路径}

### 🚀 优化建议
- {建议1：战略定位层面}
- {建议2：功能迭代层面}
- {建议3：风险规避层面}
```

**技术实现要点**：
1. NLP识别产品类型、所属行业、生命周期阶段

## 三、版本兼容性与渐进式实施路径

### 3.1 现有版本兼容性说明
**兼容基础**：所有升级完全基于现有v1.2.0版本架构，不做破坏性改动
| 现有模块 | 兼容性 | 改动说明 |
|----------|--------|----------|
| 用户系统 | 100%兼容 | 无改动，仅扩展用户偏好字段 |

| 邮票系统 | 100%兼容 | 无改动，仅新增即时回复消耗规则 |
| AI回复系统 | 100%兼容 | 扩展参数支持多类型回复，原有接口保持不变 |
| 所有现有数据 | 100%兼容 | 无需数据迁移，完全保留用户历史数据 |

### 3.2 渐进式实施路径
#### 3.2.1 第一阶段：基础重构（1-2个月）
**核心原则**：零影响现有功能，仅做基础扩展
| 时间 | 任务 | 影响范围 | 验收标准 |
|------|------|----------|----------|

| Week 3 | 优化单导师提示词+新增即时回复选项 | 云函数+写笔记页面 | 提示词无投资相关表述，即时回复功能可选 |

#### 3.2.2 第二阶段：多导师功能（2-3个月）
**核心原则**：新功能独立入口，不影响原有使用流程
| 时间 | 任务 | 影响范围 | 验收标准 |
|------|------|----------|----------|
| Week 5 | 多导师选择UI+圆桌会议入口 | 新增独立页面 | 支持3-5位导师选择，原有写笔记流程不变 |

#### 3.2.3 第三阶段：创新功能（3-6个月）
**核心原则**：灰度发布，小范围试点后全量上线
| 时间 | 任务 | 灰度策略 | 验收标准 |

| 阶段 | 回滚触发条件 | 回滚操作 | 回滚时间 |
|------|--------------|----------|----------|

| 第二阶段 | 新功能故障影响核心流程 | 隐藏圆桌会议入口，恢复原有云函数版本 | <30分钟 |
| 第三阶段 | 新功能用户反馈差 | 关闭对应功能入口，回滚相关代码 | <15分钟 |

## 四、界面与交互极简设计规范

### 4.1 核心设计原则
**张小龙/乔布斯极简设计标准**：
1. 无需用户教育，功能自然融入使用流程
2. 每页面最多3个核心操作，无冗余元素
3. 交互符合直觉，点击路径≤3步完成核心任务
4. 视觉层次清晰，重点内容突出

### 4.2 各页面设计规范
#### 4.2.1 导师选择器
```
设计规范：
┌─────────────────────────────────┐
│ 领域标签：投资/创业/心理学/哲学/全部  ← 顶部横向滚动标签
│                                 │
│ 导师卡片网格：2列布局            ← 卡片仅显示头像、姓名、头衔
│ □ [🤵] 查理·芒格                │
│    思维模型大师                 │
│                                 │
│ □ [🚀] 马斯克                   │
│    第一性原理大师               │
│                                 │
│ 底部操作：确定/取消              ← 仅2个按钮
└─────────────────────────────────┘
```

- 点击标签切换领域，自动筛选对应导师

## 四、技术实现难点分析

### 4.1 技术难点清单

| 难点 | 难度 | 说明 | 解决方案 |
|------|------|------|---------|
| **多导师上下文传递** | ⭐⭐⭐⭐ | 圆桌会议中每位导师需基于前文回复 | 设计上下文摘要机制，压缩前文关键信息 |
| **导师人格一致性** | ⭐⭐⭐⭐ | AI回复需保持导师独特风格 | 构建详细的导师人格模板和提示词系统 |
| **回复质量控制** | ⭐⭐⭐⭐ | 确保回复相关性、独特性、深度 | 设计质量评估机制，低于阈值重新生成 |
| **上下文长度限制** | ⭐⭐⭐ | 多轮对话上下文超长 | 动态摘要策略，保留关键信息 |
| **导师排序算法** | ⭐⭐⭐ | 智能排序提升讨论质量 | 基于领域、思维深度、角色互补设计规则 |
| **想法解构引擎** | ⭐⭐⭐⭐⭐ | 思想孵化器核心功能 | NLP分析+规则引擎+知识图谱 |

### 4.2 核心技术方案

#### 4.2.1 圆桌讨论提示词系统（v2.0）
**参考来源**：[圆桌讨论Prompt - 开源版本](https://gist.github.com/bill20232033cc/496d9e313049c79be7ee301b521e5764)
**版本**：v2.0 - 2026-03-17

**设计理念**：构建以"求真"为目标的结构化对话框架，由主持人引导不同思想代表进行深度对话，每轮总结生成视觉化思考框架(ASCII Chart)，通过"主动质询"与"协同共建"，对用户议题进行协同探索。

**核心原则**：
- 框架性质：建设性(constructive)
- 主持人功能：元认知引导(meta-cognitive)
- 角色原型：代表性人物(representative-figure)
- 过程流程：辩证式(dialectical)
- 交互类型：战略性行动(strategic-action)
- 输出目标：知识网络(knowledge-network)
- 角色目标：求真(truth-seeking)

**系统架构**：
```
圆桌讨论系统
├── 主持人模块
│   ├── 议题理解
│   ├── 角色推荐

│   └── 总结输出
├── 角色模块
│   ├── 导师人格模板
│   ├── 发言逻辑生成
│   ├── 观点表达风格
│   └── 回应规则
├── 对话管理模块
│   ├── 上下文维护
│   ├── 发言排序
│   ├── 质量控制
│   └── 冲突调解
└── 输出模块
    ├── 讨论稿结构化生成
    ├── ASCII思考框架可视化
    ├── 核心观点提炼
    └── 行动建议输出
```

## 八、风险分析与应对策略

### 8.1 风险识别

| 风险项 | 风险等级 | 详细描述 | 发生概率 |
|--------|---------|---------|---------|
| **AI回复质量不稳定** | 🔴 高 | AI生成的回复质量参差不齐 | 高 |
| **导师人格不一致** | 🟡 中 | AI无法准确模拟导师风格 | 中 |
| **上下文长度限制** | 🟡 中 | 多轮对话上下文超长 | 中 |
| **API成本超预算** | 🟡 中 | 用户量增长导致API费用激增 | 中 |
| **用户接受度低** | 🟡 中 | 用户不习惯多导师交互模式 | 中 |
| **技术实现延期** | 🟡 中 | 复杂功能开发周期延长 | 中 |
| **合规风险** | 🟢 低 | 内容合规问题 | 低 |

### 8.2 风险应对策略

#### 8.2.1 AI回复质量不稳定
**应对措施**：
1. 设计质量评估机制，低于阈值重新生成
2. 构建详细的导师提示词模板
3. 引入人工审核机制（初期）
4. 收集用户反馈持续优化

#### 8.2.2 导师人格不一致
**应对措施**：
1. 构建详细的导师人格模板
2. 设计人格一致性检查机制
3. 定期评估和优化提示词
4. 引入用户反馈机制

#### 8.2.3 API成本超预算
**应对措施**：
1. 设计缓存策略，减少重复调用
2. 优化提示词长度，降低token消耗
3. 设置用户调用频率限制
4. 监控API使用量，及时预警

#### 8.2.4 用户接受度低
**应对措施**：
1. 进行用户测试，收集反馈
2. 设计引导教程，降低学习成本
3. 提供多种交互模式选择
4. 持续优化用户体验
