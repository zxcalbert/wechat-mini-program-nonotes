# 分页加载与缓存机制技术方案

**制定日期**：2026-03-10  
**研究对象**：index页面笔记卡片加载优化  
**核心目标**：提升首屏加载速度，优化用户体验

---

## 一、现状分析

### 1.1 当前加载逻辑

```javascript
async fetchLetters() {
  // 1. 先count总记录数
  const countRes = await db.collection('letters').where().count()
  const total = countRes.total
  
  // 2. 循环查询所有页
  for (let i = 0; i < pageCount; i++) {
    const pageResult = await query(pageSize, skip)
    allData.push(...pageResult.data)
  }
  
  // 3. 一次性渲染所有数据
  this.setData({ letters: allData })
}
```

### 1.2 问题分析

| 问题 | 影响 |
|------|------|
| **一次性加载所有数据** | 笔记数量>50条时，首屏加载速度慢 |
| **多轮网络请求** | 循环查询多个页面，网络耗时高 |
| **无缓存机制** | 每次进入页面都重新请求，用户体验差 |
| **无加载状态** | 加载过程中页面空白，用户感知差 |

### 1.3 性能瓶颈分析

| 指标 | 当前值（50条笔记） | 优化目标 |
|------|-------------------|---------|
| 首屏加载时间 | 2-3秒 | <1秒 |
| 网络请求次数 | 3次（count + 2页数据） | 1次（首屏10条） |
| 白屏时间 | 1-2秒 | <0.5秒 |
| 内存占用 | 高（所有数据同时存在） | 低（只加载当前可见数据） |

## 二、分页加载与懒加载技术方案

### 2.1 分页加载设计

#### 2.1.1 分页参数

```javascript
data: {
  // 分页相关
  currentPage: 1,      // 当前页码
  pageSize: 10,        // 每页10条
  total: 0,            // 总记录数
  hasMore: true,       // 是否有更多数据
  isLoadingMore: false // 是否正在加载更多
}
```

#### 2.1.2 加载流程

```
页面加载
    ↓
加载第一页（10条）
    ↓
渲染首屏
    ↓
用户向下滚动
    ↓
滚动到页面底部？
    ↓ 是 ↓ 否
加载下一页        等待用户滚动
    ↓
追加到现有数据
    ↓
重复直到加载完所有数据
```

#### 2.1.3 初始加载优化

**首屏只加载10条数据**：
- 减少首次网络请求数据量
- 缩短首屏渲染时间
- 提升用户感知速度

### 2.2 滚动懒加载实现方案

#### 2.2.1 滚动监听

使用微信小程序的`onReachBottom`页面事件：
```javascript
onReachBottom: function() {
  if (this.data.hasMore && !this.data.isLoadingMore) {
    this.loadMoreLetters()
  }
}
```

#### 2.2.2 加载状态提示

| 状态 | 提示 |
|------|------|
| 加载中 | "加载中..." |
| 加载完成 | 无提示 |
| 加载失败 | "加载失败，请下拉重试" |
| 没有更多数据 | "已加载全部笔记" |

#### 2.2.3 加载状态组件设计

在页面底部添加加载提示：
```xml
<view class="loading-footer">
  <view wx:if="{{isLoadingMore}}" class="loading-tip">
    <view class="loading-spinner"></view>
    <text>加载中...</text>
  </view>
  <view wx:elif="{{!hasMore}}" class="no-more-tip">
    <text>已加载全部笔记</text>
  </view>
</view>
```

### 2.3 下拉刷新机制

保留并优化下拉刷新：
- 下拉刷新时重置分页参数
- 重新加载第一页数据
- 清空现有数据，使用新数据
- 提示用户刷新结果

## 三、本地缓存机制研究与设计

### 3.1 微信小程序缓存能力分析

#### 3.1.1 缓存类型对比

| 缓存类型 | 存储上限 | 生命周期 | 适用场景 |
|---------|---------|---------|---------|
| **数据缓存（wx.setStorage）** | 10MB | 永久存储，除非主动删除或小程序卸载 | 笔记数据、用户配置 |
| **页面缓存** | 无明确限制 | 页面栈存在时有效 | 页面状态、滚动位置 |
| **组件缓存** | 无明确限制 | 组件实例存在时有效 | 组件内部状态 |
| **内存变量** | 受内存限制 | 小程序运行时有效 | 临时数据 |

#### 3.1.2 数据缓存特性

- 异步和同步两种API
- 每个小程序存储上限10MB
- 隔离存储，不同小程序互不影响
- 支持复杂数据类型（对象、数组）

### 3.2 缓存策略设计

#### 3.2.1 笔记数据缓存方案

**缓存key设计**：
```javascript
const CACHE_KEY = `letters_${openid}_${page}`
```

**缓存结构**：
```javascript
{
  data: [],        // 笔记列表数据
  timestamp: 0,   // 缓存时间戳
  expire: 3600    // 过期时间（秒，1小时）
}
```

#### 3.2.2 缓存更新机制

**策略：先展示缓存，后台更新**
```
页面加载
    ↓
检查是否有缓存？
    ↓ 有 ↓ 无
显示缓存数据        直接请求服务器
    ↓
后台请求最新数据
    ↓
数据有更新？
    ↓ 是 ↓ 否
更新缓存并刷新页面        不处理
```

#### 3.2.3 缓存失效策略

| 场景 | 失效处理 |
|------|---------|
| 缓存过期（超过1小时） | 删除缓存，重新请求 |
| 用户新增/修改/删除笔记 | 删除相关缓存 |
| 用户主动下拉刷新 | 删除缓存，重新请求 |
| 存储空间不足 | LRU算法删除最旧的缓存 |

### 3.3 缓存空间管理策略

#### 3.3.1 存储上限控制

- 单用户笔记缓存最多存储20页（200条）
- 总缓存占用不超过1MB
- 超过上限时使用LRU算法删除最旧的缓存

#### 3.3.2 缓存清理机制

- 定期清理过期缓存（每次启动时）
- 空间不足时自动清理
- 用户退出登录时清理所有用户相关缓存

## 四、技术实现方案

### 4.1 分页加载实现

#### 4.1.1 初始加载方法

```javascript
async fetchLetters() {
  if (this.data.isLoadingMore) return
  
  this.setData({ 
    loading: true,
    currentPage: 1,
    hasMore: true,
    letters: []
  })

  try {
    // 先尝试读取缓存
    const cacheKey = `letters_${this.data.openid}_1`
    const cache = wx.getStorageSync(cacheKey)
    
    if (cache && cache.timestamp + cache.expire * 1000 > Date.now()) {
      console.log('🔄 使用缓存数据，第一页')
      const letters = this.formatLetters(cache.data)
      this.setData({ letters, displayLetters: letters })
      
      // 后台更新数据
      this.fetchPageFromServer(1, true)
    } else {
      // 无缓存，直接请求
      await this.fetchPageFromServer(1, false)
    }
  } catch (err) {
    console.error('加载失败:', err)
    wx.showToast({ title: '加载失败', icon: 'error' })
  } finally {
    this.setData({ loading: false })
  }
}
```

#### 4.1.2 分页加载方法

```javascript
async loadMoreLetters() {
  if (!this.data.hasMore || this.data.isLoadingMore) return
  
  this.setData({ isLoadingMore: true })
  
  try {
    const nextPage = this.data.currentPage + 1
    await this.fetchPageFromServer(nextPage, false)
  } catch (err) {
    console.error('加载更多失败:', err)
    wx.showToast({ title: '加载失败', icon: 'none' })
  } finally {
    this.setData({ isLoadingMore: false })
  }
}
```

#### 4.1.3 服务器请求方法

```javascript
async fetchPageFromServer(page, backgroundUpdate = false) {
  const skip = (page - 1) * this.data.pageSize
  const limit = this.data.pageSize
  
  const pageResult = await cloudbaseUtil.query('letters', {
    where: { _openid: this.data.openid },
    orderBy: 'createTime',
    orderDirection: 'desc',
    skip: skip,
    limit: limit
  })
  
  if (pageResult.success) {
    const newLetters = this.formatLetters(pageResult.data)
    
    // 更新缓存
    const cacheKey = `letters_${this.data.openid}_${page}`
    wx.setStorageSync(cacheKey, {
      data: pageResult.data,
      timestamp: Date.now(),
      expire: 3600
    })
    
    // 后台更新时只更新数据，不重置分页
    if (backgroundUpdate && page === 1) {
      this.setData({ 
        letters: newLetters, 
        displayLetters: newLetters,
        total: pageResult.count || 0,
        hasMore: pageResult.data.length === this.data.pageSize
      })
    } else {
      const allLetters = page === 1 ? newLetters : [...this.data.letters, ...newLetters]
      
      this.setData({
        letters: allLetters,
        displayLetters: allLetters,
        currentPage: page,
        hasMore: pageResult.data.length === this.data.pageSize
      })
    }
    
    return newLetters
  }
  
  throw new Error(pageResult.error || '请求失败')
}
```

### 4.2 滚动监听实现

```javascript
onReachBottom: function() {
  if (this.data.hasMore && !this.data.isLoadingMore && !this.data.loading) {
    this.loadMoreLetters()
  }
},

onPullDownRefresh: function() {
  // 下拉刷新时清除缓存
  const cachePrefix = `letters_${this.data.openid}_`
  const info = wx.getStorageInfoSync()
  info.keys.forEach(key => {
    if (key.startsWith(cachePrefix)) {
      wx.removeStorageSync(key)
    }
  })
  
  // 重新加载
  this.fetchLetters().then(() => {
    wx.stopPullDownRefresh()
    wx.showToast({ title: '刷新成功', icon: 'success' })
  }).catch(() => {
    wx.stopPullDownRefresh()
  })
}
```

| 用户操作频繁 | 加载状态锁，防止重复请求 |
