### 2.3 T3：空内容检查逻辑冲突

#### 2.3.1 当前提交流程分析

**write.js 中的 submitLetter() 流程**：
```javascript
async submitLetter() {
  // 1. 字数检查（100字）
  if (this.data.wordCount < 100) {
    wx.showModal(...);
    return;
  }
  
  // 2. 邮票检查
  if (this.data.needReply && this.data.userStamps === 0) {
    wx.showModal(...);
    return;
  }
  
  // 3. 每日限制检查
  if (this.data.needReply) {
    const limit = await this.checkDailyLimit();
    if (!limit.canSend) {
      wx.showModal(...);
      return;
    }
  }
  
  // 4. 敏感词检查
  const sensitiveCheck = this.checkSensitiveWords(this.data.content);
  if (!sensitiveCheck.canSubmit) {
    wx.showModal(...);
    return;
  }
  
  // 5. 投资敏感词提示
  if (sensitiveCheck.message) {
    wx.showModal({
      content: sensitiveCheck.message + '\n\n是否继续提交？',
      success: (res) => {
        if (res.confirm) {
          this.doSubmit();
        }
      }
    });
  } else {
    this.doSubmit();
  }
}
```

#### 2.3.2 优化后的提交流程设计

**新流程状态机**：

```
用户点击提交
    ↓
【1】敏感词检测（提交时检测）
    ├─ 检测到高敏感词 → 弹窗提示，返回修改
    └─ 通过 → 继续
    ↓
【2】空内容检查（<10字）
    ├─ <10字 → 弹窗"内容较短，确定要保存吗？"
    │         ├─ 继续 → 继续
    │         └─ 返回 → 结束
    └─ >=10字 → 继续
    ↓
【3】邮票检查（仅 needReply=true）
    ├─ 邮票=0 → 弹窗提示购买
    └─ 邮票>0 → 继续
    ↓
【4】每日限制检查（仅 needReply=true）
    ├─ 超限 → 弹窗提示明日再来
    └─ 未超限 → 继续
    ↓
【5】投资敏感词提示（如有）
    ├─ 有提示 → 弹窗"温馨提示：...是否继续提交？"
    │         ├─ 继续 → doSubmit()
    │         └─ 返回 → 结束
    └─ 无提示 → doSubmit()
```

#### 2.3.3 优化后的代码实现

```javascript
async submitLetter() {
  const content = this.data.content.trim();
  
  // 【1】敏感词检测（第一步）
  const sensitiveCheck = this.checkSensitiveWords(content);
  if (!sensitiveCheck.canSubmit) {
    wx.showModal({
      title: '内容包含敏感词',
      content: sensitiveCheck.message,
      showCancel: false,
      confirmText: '我知道了'
    });
    return;
  }
  
  // 【2】空内容检查
  if (content.length < 10) {
    wx.showModal({
      title: '内容较短',
      content: '您的内容较短，确定要保存吗？',
      confirmText: '继续保存',
      cancelText: '返回修改',
      success: (res) => {
        if (res.confirm) {
          this.continueSubmit(sensitiveCheck);
        }
      }
    });
    return;
  }
  
  // 内容正常，继续
  this.continueSubmit(sensitiveCheck);
},

async continueSubmit(sensitiveCheck) {
  // 【3】邮票检查
  if (this.data.needReply && this.data.userStamps === 0) {
    wx.showModal({
      title: '邮票不足',
      content: '需要购买邮票才能请求大师回信',
      success: (res) => {
        if (res.confirm) {
          wx.navigateTo({ url: '/pages/stamps/stamps' });
        }
      }
    });
    return;
  }
  
  // 【4】每日限制检查
  if (this.data.needReply) {
    const limit = await this.checkDailyLimit();
    if (!limit.canSend) {
      wx.showModal({
        title: '每日寄信次数已用完',
        content: '今天已寄信6次，大师需要时间深思熟虑，明天再来吧',
        showCancel: false
      });
      return;
    }
  }
  
  // 【5】投资敏感词提示
  if (sensitiveCheck.message) {
    wx.showModal({
      title: '温馨提示',
      content: sensitiveCheck.message + '\n\n是否继续提交？',
      confirmText: '继续提交',
      cancelText: '修改内容',
      success: (res) => {
        if (res.confirm) {
          this.doSubmit();
        }
      }
    });
  } else {
    this.doSubmit();
  }
},

// doSubmit() 保持基本不变，但移除字数检查
async doSubmit() {
  const mentor = this.data.needReply ? this.data.mentors[this.data.mentorIndex] : null;
  const mood = '由AI推断';
  const content = this.data.content;
  // ...
}
```

## 五、合规风险深入研究

### 5.1 C1：敏感词检测合规性

#### 5.1.1 微信小程序内容安全规范研究

**微信小程序官方文档参考**（需要查阅最新版）：

| 规范项 | 要求 | 是否强制 |
|-------|------|---------|
| **内容安全审核** | 建议对用户生成内容进行审核 | ⚠️ 推荐 |
| **msg_sec_check接口** | 提供内容安全检测接口 | ✅ 可选 |
| **实时检测要求** | 未明确要求必须实时检测 | ❌ 无强制 |
| **违规处理** | 需建立违规内容处理机制 | ⚠️ 推荐 |

**关键结论**：
- ✅ 实时检测**不是**强制要求
- ✅ 提交时检测**满足**合规要求
- ✅ 但需要建立违规内容处理机制

#### 5.1.2 建议的合规方案

**方案 A（推荐）：提交时检测 + 违规处理机制**

1. **保持提交时检测**
   - 在submitLetter()第一步进行检测
   - 高敏感词阻止提交
   - 投资敏感词提示用户

2. **建立违规内容处理机制**（补充）
   - 记录违规内容到数据库
   - 定期人工复审
   - 建立用户违规行为记录

3. **可选功能**（如需要）
   - 提供"预览检测"按钮
   - 用户可主动点击检测
   - 不强制实时检测
