# 云函数部署与环境配置指南

**文档版本**：v1.0  
**创建日期**：2026-02-28  
**问题**：部署云函数时提示"请在编辑器云函数根目录（cloudfunctionRoot）选择一个云环境"

---

## 一、问题分析

### 1.1 错误原因

微信开发者工具在部署云函数前，需要先在项目中选择一个已创建的云开发环境。如果没有选择云环境或云环境未创建，就会出现此错误。

### 1.2 前提条件

- ✅ 已注册微信小程序账号
- ✅ 已开通云开发服务
- ✅ 项目已配置云开发

---

## 二、云环境创建步骤

### 2.1 方式一：在微信开发者工具中创建云环境

#### 步骤1：打开云开发控制台

1. 在微信开发者工具顶部菜单栏，点击 **"云开发"** 按钮
2. 如果是首次打开，会弹出云开发开通引导

#### 步骤2：创建云环境

1. 点击 **"开通"** 或 **"新建环境"**
2. 填写环境名称（如：`invest-diary-dev`）
3. 选择基础版（免费版足够开发使用）
4. 点击 **"确定"**

#### 步骤3：等待环境初始化

- 环境创建通常需要1-2分钟
- 等待状态变为"运行中"

---

### 2.2 方式二：在微信公众平台创建云环境

#### 步骤1：登录微信公众平台

1. 访问 https://mp.weixin.qq.com/
2. 使用小程序管理员账号登录

#### 步骤2：进入云开发控制台

1. 左侧菜单选择 **"开发"** → **"云开发"**
2. 点击 **"开通"** 或 **"新建环境"**

#### 步骤3：创建环境

1. 填写环境名称
2. 选择计费方式
3. 确认创建

---

## 三、在微信开发者工具中选择云环境

### 3.1 配置步骤

#### 步骤1：打开云开发设置

1. 在微信开发者工具顶部，点击 **"云开发"** 按钮
2. 在弹出的云开发控制台中，确认环境已创建

#### 步骤2：在项目中选择云环境

1. 在微信开发者工具左侧目录树，右键点击 `cloudfunctions` 文件夹
2. 选择 **"当前环境"** → 选择刚才创建的云环境
3. 或者点击工具栏的 **"云环境"** 下拉框选择环境

#### 步骤3：验证环境选择

1. 查看 `cloudfunctions` 文件夹图标是否变为云朵图标
2. 查看状态栏是否显示当前选择的云环境名称

---

### 3.2 配置project.config.json（可选）

如果需要在配置文件中指定环境，可以修改 `project.config.json`：

```json
{
  "miniprogramRoot": "miniprogram/",
  "cloudfunctionRoot": "cloudfunctions/",
  "cloudbaseRoot": "cloudbase/",
  "setting": {
    "urlCheck": true,
    "es6": true
  },
  "appid": "wxbaa7aa9905799960",
  "projectname": "quickstart-wx-cloud",
  "libVersion": "3.7.1",
  "cloudfunctionTemplateRoot": "cloudfunctionTemplate/",
  "condition": {},
  "compileType": "miniprogram",
  "srcMiniprogramRoot": "miniprogram/",
  "packOptions": {},
  "editorSetting": {}
}
```

## 四、云函数部署步骤

### 4.1 部署getMentorRules云函数

#### 步骤1：确认云环境已选择

- 确保已按上述步骤选择云环境
- `cloudfunctions` 文件夹显示云朵图标

#### 步骤2：部署云函数

1. 在左侧目录树中，右键点击 `cloudfunctions/getMentorRules` 文件夹
2. 选择 **"上传并部署：云端安装依赖"**
3. 等待部署完成（通常需要1-2分钟）

#### 步骤3：验证部署成功

1. 部署完成后，控制台会显示"上传成功"
2. 点击顶部 **"云开发"** 按钮
3. 进入 **"云函数"** 标签页
4. 确认 `getMentorRules` 函数在列表中

---

### 4.2 云函数测试

#### 方式一：在云开发控制台测试

1. 进入云开发控制台 → **云函数**
2. 找到 `getMentorRules`，点击 **"测试"**
3. 点击 **"调用测试"**
4. 查看返回结果是否为：
   ```json
   {
     "success": true,
     "data": { ... }
   }
   ```

#### 方式二：在小程序中测试

1. 在 `write.js` 的 `loadMentorRules()` 方法中添加 `console.log`
2. 编译运行小程序
3. 打开调试器Console
4. 查看是否成功加载规则库

## 五、app.js云开发初始化配置

### 5.1 当前配置

项目中的 `app.js` 已正确配置云开发初始化：

```javascript
wx.cloud.init({
  traceUser: true
});
```

**说明**：不显式指定 `env` 参数时，会使用当前选择的云环境。

### 5.2 如需指定环境

如果需要指定特定环境，可以修改为：

```javascript
wx.cloud.init({
  env: 'your-env-id',  // 替换为你的环境ID
  traceUser: true
});
```

**获取环境ID**：
1. 进入云开发控制台
2. 点击 **"设置"** → **"环境设置"**
3. 复制环境ID

## 六、常见问题排查

### 6.1 问题：找不到云开发按钮

**解决方案**：
- 确认使用的是最新版微信开发者工具
- 确认小程序AppID已正确配置
- 确认已在微信公众平台开通云开发

### 6.2 问题：云环境创建失败

**解决方案**：
- 检查账号是否已实名认证
- 检查是否已有多个环境（免费版限2个环境）
- 尝试使用不同的环境名称

### 6.3 问题：云函数部署超时

**解决方案**：
- 检查网络连接
- 尝试"上传并部署：不安装依赖"，然后手动在云端安装
- 检查云函数代码是否有语法错误

### 6.4 问题：云函数调用失败

**解决方案**：
- 检查云函数是否已成功部署
- 检查云函数名称是否正确
- 查看云开发控制台的云函数日志
- 确认小程序AppID与云环境匹配
