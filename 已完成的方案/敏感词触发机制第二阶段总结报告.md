# 敏感词触发机制第二阶段总结报告

## 一、项目概述

### 1.1 项目背景
基于微信小程序平台对投资理财类应用的合规要求，需要实现敏感词触发机制，确保：
- 不生成暴力、涉黄、宗教信仰相关内容
- 不生成关于投资标的相关具体估值分析和投资理财建议
### 1.2 项目目标
**主要目标**：实现敏感词检测、分级触发及相应处理策略
**具体目标**:
1. 实现敏感词检测功能
2. 实现分级触发机制
3. 提供良好的用户体验
4. 满足合规要求
### 1.3 项目范围
**前端**：miniprogram/pages/write/write.js
**云函数**：cloudfunctions/replyToLetter/index.js
**工具类**：miniprogram/utils/sensitiveWordUtil.js

### 2.2 敏感词工具类
**文件**：[miniprogram/utils/sensitiveWordUtil.js](file:///Users/bill/编程/invest-diary/miniprogram/utils/sensitiveWordUtil.js)
**功能**：
- `hasSensitiveWord(text)` - 检测是否包含敏感词
- `detect(text)` - 获取敏感词详情(包含哪些词、哪些分类)
- `filter(text)` - 过滤敏感词(替换为***)
**敏感词库**：
- **暴力**：杀人、暴力、恐怖,袭击,爆炸,枪支、武器,血腥,残忍,虐待,杀戮,谋杀,伤害,残害
- **涉黄**:色情、淫秽,性交易,嫖娵,卖淫,裸聊,成人,黄色,低俗,下流,猥琐,淫秽物品
- **投资建议**：买入,卖出,推荐,估值,收益率,回报率,目标价,股价,基金净值,投资建议,理财规划;增持,减持,抄底,逃顶,加仓,减仓

### 2.4 警告机制优化
**分析报告**：[敏感词警告机制优化方案分析报告.md](file:///Users/bill/编程/invest-diary/敏感词警告机制优化方案分析报告.md)
**最终决策**：采用方案二(确认对话框)
**推荐理由**:
1. 用户选择权:用户有明确的选择,体验更好
2. 误操作风险低:用户必须处理弹窗,不会误提交
3. 酷读充分:用户有足够时间阅读警告内容
4. 符合设计原则:强制确认可避免后悔
**详细内容**:参见警告机制优化分析报告
### 2.5 云函数集成
**文件**：[cloudfunctions/replyToLetter/index.js](file:///Users/bill/编程/invest-diary/cloudfunctions/replyToLetter/index.js)
**新增功能**：
- 添加敏感词库(sensitiveWords对象)
- 实现`detectSensitiveWords(text)` 函数
- 实现`processReply(replyContent)` 函数
- 在AI回复生成后调用`processReply()`处理理
**处理逻辑**:
| 敏感度 | 检测结果 | 处理方式 |
|--------|---------|---------|
| 🔴 高 | 暴力、涉黄 | 返回通用回复："感谢你的来信。由于内容合规性要求，我无法针对这个话题给出具体回复。建议你从更宏观的角度思考问题，关注原则和方法论，而不是具体的标的或建议。" |
| 🟡 中 | 投资建议:添加免责声明："⚠️ 免责声明：以上内容仅供参考，不构成任何投资建议。投资有风险,决策需谨慎。"
            🟢 无 | 无敏感词 | 直接返回原回复
