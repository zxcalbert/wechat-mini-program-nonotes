## 一、方案概述

### 1.1 目标

实现敏感词检测、分级触发及相应处理策略，确保：
- 不生成暴力、涉黄、宗教信仰相关内容
- 不生成关于投资标的相关具体估值分析和投资理财建议
- 提供良好的用户体验，避免过度打扰

### 1.2 范围

| 检测点 | 时机 | 位置 |
|-------|------|------|
| 用户输入检测 | 提交前检测 | write 页面 |
| AI回复检测 | 回复生成后检测 | 云函数 replyToLetter |

### 1.3 重要决策

**最终决策**：移除输入实时监测，采用提交前检测

**决策理由**：
1. 产品定位：个人工具，不是内容审核平台
2. 用户体验：实时监测打扰用户思考，不符合"堆肥"理念
3. 隐私敏感：个人笔记频繁检测，隐私感降低
4. 风险控制：核心风险在AI回复，不在用户输入
5. 成本效益：实时监测成本高，收益低

**详细分析**：参见 `敏感词触发机制-输入实时监测分析报告.md`

## 三、检测策略

### 3.1 用户输入检测

**时机**：提交前检测（点击提交按钮时检测一次）

**策略**：
- 检测到高敏感度敏感词（暴力、涉黄）时，阻止提交
- 检测到中敏感度敏感词（投资建议）时，警告提示但允许提交

### 3.2 AI回复检测

**时机**：云函数生成AI回复后

**策略**：
- 检测到高敏感度敏感词时，替换为通用回复
- 检测到中敏感度敏感词时，添加免责声明

## 四、实现方案

### 4.1 工具类设计

**文件**：`miniprogram/utils/sensitiveWordUtil.js`

```javascript
class SensitiveWordUtil {
  constructor() {
    this.words = {
      violence: [
        '杀人', '暴力', '恐怖', '袭击', '爆炸', '枪支', '武器',
        '血腥', '残忍', '虐待', '杀戮', '谋杀', '伤害', '残害'
      ],
      porn: [
        '色情', '淫秽', '性交易', '嫖娼', '卖淫', '裸聊',
        '成人', '黄色', '低俗', '下流', '猥琐', '淫秽物品'
      ],
      investment: [
        '买入', '卖出', '推荐', '估值', '收益率', '回报率',
        '目标价', '股价', '基金净值', '投资建议', '理财规划',
        '增持', '减持', '抄底', '逃顶', '加仓', '减仓'
      ]
    };
  }

  /**
   * 检测文本是否包含敏感词
   * @param {string} text - 待检测文本
   * @returns {boolean} - 是否包含敏感词
   */
  hasSensitiveWord(text) {
    if (!text) return false;

    for (const category in this.words) {
      for (const word of this.words[category]) {
        if (text.includes(word)) {
          return true;
        }
      }
    }
    return false;
  }

  /**
   * 获取敏感词详情
   * @param {string} text - 待检测文本
   * @returns {object} - 包含检测到的敏感词
   */
  detect(text) {
    if (!text) {
      return {
        hasSensitive: false,
        words: [],
        categories: []
      };
    }

    const foundWords = [];
    const foundCategories = [];

    for (const category in this.words) {
      for (const word of this.words[category]) {
        if (text.includes(word)) {
          if (!foundWords.includes(word)) {
            foundWords.push(word);
          }
          if (!foundCategories.includes(category)) {
            foundCategories.push(category);
          }
        }
      }
    }

    return {
      hasSensitive: foundWords.length &gt; 0,
      words: foundWords,
      categories: foundCategories,
      isHighSensitive: foundCategories.some(c =&gt; c === 'violence' || c === 'porn'),
      isInvestment: foundCategories.includes('investment')
    };
  }

  /**
   * 过滤敏感词（替换为***）
   * @param {string} text - 待过滤文本
   * @returns {string} - 过滤后的文本
   */
  filter(text) {
    if (!text) return text;

    let result = text;
    for (const category in this.words) {
      for (const word of this.words[category]) {
        const regex = new RegExp(word, 'g');
        result = result.replace(regex, '***');
      }
    }
    return result;
  }
}

module.exports = new SensitiveWordUtil();
```

## 五、前端实现

### 5.1 写笔记页面集成

**文件**：`miniprogram/pages/write/write.js`

```javascript
const sensitiveWordUtil = require('../../utils/sensitiveWordUtil');

Page({
  data: {
    // ...
    hasSensitiveWarning: false,
    sensitiveWarning: ''
  },

  onInput(e) {
    const wordCount = e.detail.value.length;
    this.setData({
      content: e.detail.value,
      wordCount: wordCount,
      canSend: wordCount &gt;= 100
    });
  },

  checkSensitiveWords(text) {
    const result = sensitiveWordUtil.detect(text);

    if (result.hasSensitive) {
      if (result.isHighSensitive) {
        return {
          canSubmit: false,
          message: '您的内容包含敏感词，请修改后再提交。'
        };
      } else if (result.isInvestment) {
        return {
          canSubmit: true,
          message: '注意：您的内容包含投资相关词汇，AI回复不会提供具体投资建议。'
        };
      }
    }

    return {
      canSubmit: true,
      message: ''
    };
  },

  async submitLetter() {
    // ... 其他验证 ...

    const sensitiveCheck = this.checkSensitiveWords(this.data.content);

    if (!sensitiveCheck.canSubmit) {
      wx.showModal({
        title: '内容包含敏感词',
        content: sensitiveCheck.message,
        showCancel: false
      });
      return;
    }

    if (sensitiveCheck.message) {
      this.setData({
        hasSensitiveWarning: true,
        sensitiveWarning: sensitiveCheck.message
      });
    }

    // ... 提交逻辑 ...
  }
});
```

## 八、风险与应对

### 8.1 误杀风险

**风险**：敏感词检测可能误杀正常内容

**应对**：
- 使用较宽松的敏感词库
- 提供用户反馈机制
- 定期审核和优化敏感词库

### 8.2 漏杀风险

**风险**：敏感词检测可能漏掉违规内容

**应对**：
- 持续更新敏感词库
- 建立用户举报机制
- 定期人工审核

### 8.3 性能风险

**风险**：敏感词检测可能影响性能

**应对**：
- 使用防抖（debounce）减少检测频率
- 优化检测算法
- 在云函数中异步处理
