## 一、项目背景与目标

### 1.1 项目背景

基于张小龙产品观的评估，当前写笔记页面存在以下问题：
- 功能过于复杂，用户决策负担重
- 界面元素过多，视觉噪音大
- 不符合"用完即走"和"简单就是美"的产品理念

### 1.2 优化目标

| 目标 | 说明 | 预期效益 |
|------|------|---------|
| **简化界面** | 移除不必要的UI元素 | ⭐⭐⭐⭐⭐ |
| **减少决策** | 减少用户选择步骤 | ⭐⭐⭐⭐⭐ |
| **提升体验** | 符合"用完即走"理念 | ⭐⭐⭐⭐⭐ |
| **保持合规** | 敏感词检测、免责声明 | ⭐⭐⭐⭐ |

### 1.3 核心原则

严格遵循张小龙产品观：
- ✅ 用完即走：不占用用户时间
- ✅ 以用户为中心：而非以功能为中心
- ✅ 简单就是美：追求极致的简单
- ✅ 功能克制：只做核心功能

#### 任务2.1：移除最小字数限制，添加空内容警告

**优先级**：🔴 高  
**预计周期**：2天  
**负责角色**：前端开发

**当前实现**：
- 最小字数限制100字
- 未达到限制无法提交
- canSend依赖字数检查

**优化方案**：
1. 移除最小字数限制检查
2. 添加空内容警告弹窗
3. 内容为空或<10字时，弹窗提醒用户

**技术实现**：

```javascript
// write.js - 移除字数限制，添加空内容警告
Page({
  data: {
    content: '',
    canSend: true  // 默认可发送（移除字数限制）
  },
  
  onInput(e) {
    this.setData({
      content: e.detail.value
    });
  },
  
  submitLetter() {
    const content = this.data.content.trim();
    
    // 空内容警告
    if (content.length < 10) {
      wx.showModal({
        title: '内容较短',
        content: '您的内容较短，确定要保存吗？',
        confirmText: '继续保存',
        cancelText: '返回修改',
        success: (res) => {
          if (res.confirm) {
            this.doSubmit();
          }
        }
      });
    } else {
      this.doSubmit();
    }
  },
  
  doSubmit() {
    // 实际提交逻辑
  }
});
```

**验收标准**：
- ✅ 移除最小字数限制
- ✅ 空内容（<10字）弹窗警告
- ✅ 用户可选择继续保存或返回修改
- ✅ 正常内容直接提交

```javascript
    }
  }
```

```javascript
// replyToLetter/index.js - 优化提示词
function getMentorPrompt(mentor, content) {
  return `请以${mentor}的身份回复用户的问题。

【重要约束】
1. 只基于${mentor}的公开言论和观点进行回复
2. 不要编造${mentor}没说过的话
3. 如不确定，使用更通用的表述
4. 回复必须直接、具体、有针对性
5. 字数控制在200-500字

【用户内容】
${content}

请直接回复，无需重复约束条件。`;
}

// 添加免责声明
function addDisclaimer(reply) {
  const disclaimer = '\n\n---\n*以上内容为AI模拟生成，仅供参考和启发，不代表该人物的真实观点或建议。*';
  return reply + disclaimer;
}
```
