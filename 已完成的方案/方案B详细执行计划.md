
# 夜晚模式兼容性问题修复方案 B 详细执行计划

## 一、项目概述

### 1.1 项目名称
微信小程序夜晚模式兼容性问题修复（方案 B）

### 1.2 项目背景
当前项目使用 `wx.setThemeStyle` API 实现主题切换功能，但在部分环境（特别是微信开发者工具）中该 API 不可用，导致 TypeError 错误。方案 B 选用自定义样式类切换方案，完全移除对微信原生 API 的依赖，使用 CSS 变量 + 媒体查询实现主题功能。

### 1.3 项目目标
- 彻底解决 `wx.setThemeStyle` 兼容性问题
- 实现 100% 兼容所有基础库版本
- 支持亮色/暗色主题切换
- 支持系统主题自动跟随
- 用户主题偏好持久化存储

### 1.4 项目范围
- 修改 app.wxss（全局样式）
- 修改 app.js（应用逻辑）
- 修改 app.json（配置文件）
- 修改 6 个页面（index, write, detail, stamps, login, trash）
- 更新实施计划文档

## 八、详细实施步骤

### 步骤 1：修改 app.wxss
```
1. 打开 miniprogram/app.wxss
2. 在文件顶部添加亮色主题 CSS 变量
3. 添加 .theme-light 类定义（覆盖变量）
4. 添加 .theme-dark 类定义（覆盖变量）
5. 添加 @media (prefers-color-scheme: dark) 媒体查询
6. 保存文件
```

### 步骤 2：修改 app.js
```
1. 打开 miniprogram/app.js
2. 移除 setTheme 中的 wx.setThemeStyle 调用
3. 添加 getThemeClass() 方法
4. 保留 initTheme、getTheme、toggleTheme 存储逻辑
5. 保存文件
```

### 步骤 3：修改 app.json
```
1. 打开 miniprogram/app.json
2. 移除 "darkmode": true
3. 移除 "themeLocation": "theme.json"
4. 保存文件
```

### 步骤 4：修改页面（6个）
```
对每个页面执行：
1. 打开页面的 .wxml 文件
2. 在根容器添加 class="{{themeClass}}"
3. 打开页面的 .js 文件
4. 在 onLoad 中添加获取 themeClass 的逻辑
5. 保存文件
```

### 步骤 5：测试验证
```
1. 在开发者工具中运行项目
2. 测试亮色主题显示
3. 测试暗色主题显示
4. 测试主题切换
5. 测试系统主题跟随
6. 测试用户偏好保存
7. 检查控制台无错误
```
