# 每日寄送次数限制修复报告

## 一、问题概述

### 1.1 问题描述

**问题ID**：10.3  
**问题名称**：每日寄送次数限制未生效  
**优先级**：高优先级  
**影响范围**：write页面（寄信页面）

**原问题**：如果选择需要回复，一天只能寄送2次，但是这个限制没有生效。

---

## 二、问题分析

### 2.1 根本原因

**核心问题**：`createTime` 数据类型不匹配导致查询失败

#### 问题详情

**1. 数据写入（write.js）**
```javascript
// 写入时使用 db.serverDate()
const addRes = await db.collection('letters').add({
  data: {
    mentor: mentor,
    mood: mood,
    content: content,
    status: needReply ? 'pending' : 'saved',
    needReply: needReply,
    createTime: db.serverDate(),  // 使用微信云数据库的日期对象
    replyExpectTime: needReply ? new Date(Date.now() + 18 * 60 * 60 * 1000).getTime() : null
  }
});
```

**2. 数据查询（checkDailyLimit）**
```javascript
// 查询时使用时间戳（number类型）
const startOfDay = new Date(today.getFullYear(), today.getMonth(), today.getDate());
const endOfDay = new Date(today.getFullYear(), today.getMonth(), today.getDate() + 1);

const result = await cloudbaseUtil.query('letters', {
  where: {
    _openid: this.data.openid,
    needReply: true,
    createTime: {
      $gte: startOfDay.getTime(),  // 使用时间戳（number类型）
      $lt: endOfDay.getTime()
    }
  }
});
```

#### 问题分析

| 操作 | 数据类型 | 说明 |
|------|---------|------|
| 写入 | `db.serverDate()` | 微信云数据库日期对象 |
| 查询 | `startOfDay.getTime()` | JavaScript时间戳（number） |

**问题**：微信云数据库的日期字段不能直接与JavaScript时间戳（number类型）比较，导致：
1. 查询条件不匹配
2. 查询结果总是返回0条记录
3. 每日限制检查失效
4. 用户可以无限次寄送需要回复的信件

---

## 三、修复方案

### 3.1 修复思路

使用微信云数据库的 `db.command` 构建正确的日期查询条件，使用 `Date` 对象而不是时间戳。

### 3.2 修复代码

**文件**：`/Users/bill/编程/invest-diary/miniprogram/pages/write/write.js`

**修改前**：
```javascript
async checkDailyLimit() {
  try {
    const today = new Date();
    const startOfDay = new Date(today.getFullYear(), today.getMonth(), today.getDate());
    const endOfDay = new Date(today.getFullYear(), today.getMonth(), today.getDate() + 1);

    const result = await cloudbaseUtil.query('letters', {
      where: {
        _openid: this.data.openid,
        needReply: true,
        createTime: {
          $gte: startOfDay.getTime(),
          $lt: endOfDay.getTime()
        }
      }
    });

    const totalSent = result.success ? result.data.length : 0;
    const remaining = Math.max(0, 2 - totalSent);

    return {
      canSend: remaining &gt; 0,
      remaining: remaining,
      total: totalSent
    };
  } catch (err) {
    console.error('检查每日限制失败:', err);
    return { canSend: true, remaining: 2, total: 0 };
  }
},
```

**修改后**：
```javascript
async checkDailyLimit() {
  try {
    const db = wx.cloud.database();
    const _ = db.command;
    const today = new Date();
    const startOfDay = new Date(today.getFullYear(), today.getMonth(), today.getDate());
    const endOfDay = new Date(today.getFullYear(), today.getMonth(), today.getDate() + 1);

    const result = await cloudbaseUtil.query('letters', {
      where: {
        _openid: this.data.openid,
        needReply: true,
        createTime: _.gte(startOfDay).and(_.lt(endOfDay))
      }
    });

    const totalSent = result.success ? result.data.length : 0;
    const remaining = Math.max(0, 2 - totalSent);

    return {
      canSend: remaining &gt; 0,
      remaining: remaining,
      total: totalSent
    };
  } catch (err) {
    console.error('检查每日限制失败:', err);
    return { canSend: true, remaining: 2, total: 0 };
  }
},
```

### 3.3 修改说明

| 修改项 | 说明 |
|-------|------|
| `const db = wx.cloud.database()` | 获取数据库实例 |
| `const _ = db.command` | 获取数据库命令对象 |
| `_.gte(startOfDay).and(_.lt(endOfDay))` | 使用微信云数据库的链式查询语法 |

## 六、维护建议

### 6.1 代码规范

在微信云开发中，日期查询应遵循以下规范：

```javascript
// ✅ 正确：使用 db.command 链式查询
const db = wx.cloud.database();
const _ = db.command;
const startOfDay = new Date(2024, 0, 1);
const endOfDay = new Date(2024, 0, 2);

where: {
  createTime: _.gte(startOfDay).and(_.lt(endOfDay))
}

// ❌ 错误：使用时间戳比较
where: {
  createTime: {
    $gte: startOfDay.getTime(),
    $lt: endOfDay.getTime()
  }
}
```

### 6.2 后续优化建议

1. **时区处理**：当前使用本地时间，建议统一使用UTC时间
2. **云函数验证**：在云函数中也增加每日限制检查，作为双重验证
3. **用户提示**：在界面上显示剩余寄送次数，提升用户体验

### 7.2 经验总结

1. **微信云数据库查询**：必须使用 `db.command` 构建查询条件，不能直接使用JavaScript对象语法
2. **日期类型**：微信云数据库的日期字段是特殊类型，不能与时间戳直接比较
3. **双重验证**：建议前端和云函数都进行验证，防止绕过限制
4. **测试覆盖**：对于关键功能（如成本控制），需要全面的测试覆盖
