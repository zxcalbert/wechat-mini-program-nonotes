## 二、第一阶段实施内容

### 2.1 AI生成内容提示

#### 需求描述
在回复页面添加AI生成内容提示，明确告知用户AI生成内容仅供参考，不构成任何建议。

#### 技术实现

**修改文件**：
- [detail.wxml](file:///Users/bill/编程/invest-diary/miniprogram/pages/detail/detail.wxml)
- [detail.wxss](file:///Users/bill/编程/invest-diary/miniprogram/pages/detail/detail.wxss)

**修改内容**：

**detail.wxml**：在回复卡片上方添加AI免责声明
```xml
<block wx:if="{{letter.canShowReply}}">
  <view class="ai-disclaimer">
    <text class="disclaimer-text">⚠️ AI 生成内容仅供参考，不构成任何建议</text>
  </view>
  <view class="reply-card">
    <!-- 原有回复卡片内容 -->
  </view>
</block>
```

**detail.wxss**：添加免责声明样式
```css
.ai-disclaimer {
  background-color: #fff3cd;
  border: 1rpx solid #ffc107;
  border-radius: 8rpx;
  padding: 20rpx 30rpx;
  margin-bottom: 30rpx;
}

.disclaimer-text {
  font-size: 26rpx;
  color: #856404;
  line-height: 1.5;
}
```

#### 设计说明

| 设计元素 | 说明 |
|---------|------|
| 背景色 | #fff3cd（浅黄色），符合警告提示的设计规范 |
| 边框色 | #ffc107（警告色），与背景色协调 |
| 文字色 | #856404（深黄色），确保可读性 |
| 图标 | ⚠️（警告图标），直观传达提示含义 |
| 位置 | 回复卡片上方，确保用户在阅读回复前看到提示 |

### 2.2 用户隐私提示

#### 需求描述
确保用户在注册/使用时看到隐私协议提示。

#### 现状分析
经检查，登录页面（login.wxml）已包含基本的隐私提示：

```xml
<view class="privacy-notice">
  <text class="notice-title">隐私说明</text>
  <text class="notice-text">我们需要获取你的微信昵称和头像用于识别身份。你的所有信件和回复内容将被严格保护，仅限你本人可以查看。</text>
</view>
```

**结论**：用户隐私提示已存在，无需额外修改。

## 五、后续阶段规划

### 5.1 第二阶段：敏感词触发机制

**预计工时**：2-3天

**任务清单**：
1. 建立敏感词库（暴力、涉黄、宗教、投资建议等）
2. 在用户输入时进行敏感词检测
3. 在AI回复生成后进行敏感词检测
4. 实现敏感词过滤或替换逻辑
5. 处理敏感词触发后的用户提示

**风险点**：
- 误杀风险：敏感词检测可能误杀正常内容
- 漏杀风险：敏感词检测可能漏掉违规内容
- 性能风险：敏感词检测可能影响AI回复生成速度

### 5.2 第三阶段：数据库内容加密

**预计工时**：2-3天

**任务清单**：
1. 研究微信云开发的加密功能
2. 设计加密方案（权衡安全性和性能）
3. 实现数据加密存储
4. 实现数据解密读取
5. 测试加密后的查询性能

**风险点**：
- 性能风险：数据库加密可能影响查询性能
- 技术风险：加密实现可能存在安全漏洞
- 兼容性风险：加密可能影响现有功能
