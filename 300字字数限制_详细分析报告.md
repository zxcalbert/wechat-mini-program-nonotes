## 一、300字字数限制的设计考量

### 1.1 用户体验因素

| 考量维度 | 分析 | 依据 |
|---------|------|------|
| **阅读体验** | 300字是用户快速阅读的最佳长度 | 研究表明，手机屏幕上300字的阅读时间约15-20秒 |
| **注意力保持** | 300字能保持用户注意力不分散 | 超过400字用户开始跳读，关键信息容易被忽略 |
| **用户预期** | 符合"碎片化阅读"习惯 | 用户期望简洁、有针对性的回复，而非长篇大论 |
| **复用性** | 300字内容便于回顾和引用 | 用户多次查看笔记时，300字易于快速获取核心信息 |

### 1.2 信息传递效率因素

| 考量维度 | 分析 | 依据 |
|---------|------|------|
| **核心聚焦** | 300字强迫AI聚焦核心观点 | 避免泛泛而谈，确保每条回复都有1-2个核心观点 |
| **原则传递** | 300字足够传递投资原则 | 投资原则通常简洁有力，不需要冗长解释 |
| **认知负担** | 300字降低用户认知负担 | 用户在投资思考时已耗费精力，回复应简洁 |
| **可操作性** | 300字建议便于用户执行 | 具体、可操作的建议通常在200-300字内能说清楚 |

### 1.3 系统性能因素

| 考量维度 | 分析 | 依据 |
|---------|------|------|
| **API成本** | 300字降低Token消耗 | DeepSeek API按Token计费，300字约200-250 Token |
| **响应速度** | 300字生成速度更快 | 生成500字比300字慢约30-50% |
| **并发能力** | 300字提升系统吞吐量 | 单位时间内能处理更多用户请求 |
| **存储成本** | 300字降低数据库存储 | 每条回复少200字，长期积累节省可观 |

### 1.4 产品理念因素

| 考量维度 | 分析 | 依据 |
|---------|------|------|
| **"堆肥"理念** | 简洁回复鼓励用户深度思考 | 不提供所有答案，让用户自己"堆肥"沉淀智慧 |
| **flomo借鉴** | flomo强调"少即是多" | 极简设计，用户自己思考而非依赖AI |
| **长期主义** | 简洁回复符合长期主义 | 投资是长期过程，不需要即时满足 |
| **反依赖** | 简洁回复避免用户过度依赖AI | 培养用户独立思考能力，这是投资的核心 |

## 二、300字内容精简策略与替代方案

### 2.1 官方建议的内容精简策略

#### 策略1：结构约束法

**回复结构**（3段式，约250-300字）：
```
第1段（约80字）：核心原则
- 直接点明1个核心原则
- 用大师的典型表达

第2段（约120字）：具体建议
- 针对用户问题给出1-2条具体建议
- 建议必须可操作

第3段（约80字）：总结启发
- 升华到投资理念层面
- 鼓励用户继续思考
```

**示例**（查理·芒格风格）：
```
正如我常说的，反过来想，总是反过来想。当你遇到困难时，先想想如何避免失败，而不是如何成功。

针对你提到的市场波动问题，我建议你：1. 列出可能导致亏损的3个因素；2. 制定应对每个因素的预案。记住，避免错误比追求完美更重要。

投资不是短跑，而是马拉松。保持理性，避免短期情绪干扰，你会走得更远。继续思考，写下你的想法。（267字）
```

---

#### 策略2：关键词提取法

**步骤**：
1. 从用户内容中提取3个核心关键词
2. 围绕这3个关键词展开
3. 每个关键词用1-2句话说明
4. 最后用1句话总结

**示例**：
```
关键词：市场波动、安全边际、长期持有

市场波动是常态，不要被短期波动影响决策。安全边际是投资的生命线，永远不要支付过高价格。长期持有优秀企业，让时间成为你的朋友。

记住这三点，你就能在投资中保持理性。（142字）
```

---

#### 策略3：问题聚焦法

**步骤**：
1. 识别用户的1个核心问题
2. 直接回答这个问题
3. 给出1个具体的行动建议
4. 用大师的1条原则支撑

**示例**：
```
你的核心问题是：如何面对市场波动？

我的建议是：不要看短期价格，看企业内在价值。

正如巴菲特所说："时间是优秀企业的朋友，平庸企业的敌人。"（112字）
```

### 2.2 替代方案：灵活字数机制

如果担心300字不够，可以考虑**灵活字数机制**：

#### 方案A：分级字数（推荐）

| 级别 | 字数范围 | 触发条件 | 说明 |
|-----|---------|---------|------|
| **标准** | 200-300字 | 默认 | 大多数情况适用 |
| **详细** | 300-500字 | 用户明确要求详细回复 | 用户可选择"详细模式" |
| **极简** | 100-200字 | 用户选择"快速模式" | 快速获取核心观点 |

**前端实现**：
- 默认：200-300字
- 复选框："需要详细回复" → 300-500字
- 复选框："快速获取观点" → 100-200字

---

#### 方案B：渐进式揭示

**机制**：
1. 先显示300字核心回复
2. 底部添加"查看更多"按钮
3. 点击后显示额外200-300字补充内容

**优点**：
- 默认简洁，符合设计理念
- 用户可选择是否深入
- 兼顾简洁和详细需求

---

#### 方案C：字数自适应

**机制**：
- AI根据用户内容复杂度自动调整
- 简单问题：200字
- 中等问题：300字
- 复杂问题：400-500字
- **上限**：500字

**实现**：
- 在提示词中让AI评估复杂度
- 允许AI在合理范围内调整
- 严格执行500字上限

---

### 2.3 内容质量保证措施

即使字数有限，也要保证内容质量：

| 措施 | 说明 |
|-----|------|
| **必须包含核心原则** | 至少1条大师的核心原则 |
| **必须具体** | 避免"你应该..."，要说"你可以列出3个..." |
| **必须可操作** | 建议必须是用户能实际执行的 |
| **必须有启发** | 结尾鼓励用户继续思考 |
| **检查清单** | 生成回复后检查是否符合以上4点 |

### 3.4 字数统计与截断规则详解

#### 中文字数统计方法

| 方法 | 说明 | 准确度 |
|-----|------|-------|
| **按字符数** | 直接 `text.length` | ⭐⭐ 简单但不准确（标点也算） |
| **按汉字数** | 正则 `/[\u4e00-\u9fa5]/g` | ⭐⭐⭐⭐ 只算汉字，较准确 |
| **按词数** | 分词后统计 | ⭐⭐⭐⭐⭐ 最准确，但需要分词库 |

**推荐实现**（折中方案）：

```javascript
function countChineseWords(text) {
  // 匹配汉字
  const chineseChars = text.match(/[\u4e00-\u9fa5]/g) || [];
  // 匹配英文单词
  const englishWords = text.match(/[a-zA-Z]+/g) || [];
  
  // 粗略估算：1个英文单词≈1.5个汉字
  return chineseChars.length + Math.round(englishWords.length * 1.5);
}

function truncateByChineseWords(text, maxWords = 300) {
  const currentCount = countChineseWords(text);
  if (currentCount <= maxWords) {
    return text;
  }
  
  // 二分查找找到截断点
  let low = 0, high = text.length;
  let bestTruncated = text.substring(0, Math.floor(text.length * 0.7));
  
  while (low <= high) {
    const mid = Math.floor((low + high) / 2);
    const candidate = text.substring(0, mid);
    const count = countChineseWords(candidate);
    
    if (count <= maxWords) {
      bestTruncated = candidate;
      low = mid + 1;
    } else {
      high = mid - 1;
    }
  }
  
  // 找到最后一个完整句子
  const lastPunctuation = Math.max(
    bestTruncated.lastIndexOf('。'),
    bestTruncated.lastIndexOf('！'),
    bestTruncated.lastIndexOf('？')
  );
  
  if (lastPunctuation > 0) {
    return bestTruncated.substring(0, lastPunctuation + 1);
  }
  
  return bestTruncated;
}
```

### 3.5 异常处理机制

| 异常场景 | 处理策略 |
|---------|---------|
| AI生成超过500字 | 强制截断到300字，保留完整句子 |
| AI生成不足100字 | 提示"继续补充"，或补充默认内容 |
| 截断后语义不完整 | 优先保留核心原则，牺牲次要细节 |
| API返回为空 | 使用规则生成器作为备用方案 |

**异常处理代码**：

```javascript
async function generateReplyWithFallback(mentor, mood, content) {
  try {
    const systemPrompt = getMentorPrompt(mentor, mood, content);
    let reply = await callDeepSeekAPI(systemPrompt, content);
    
    // 字数校验
    const wordCount = countChineseWords(reply);
    if (wordCount > 300) {
      console.log(`AI生成了${wordCount}字，超过300字，进行截断`);
      reply = truncateByChineseWords(reply, 300);
    } else if (wordCount < 100) {
      console.log(`AI生成了${wordCount}字，不足100字，使用规则生成器补充`);
      const ruleReply = generateSmartReply(mentor, mood, content);
      // 合并AI回复和规则回复
      reply = reply + '\n\n' + ruleReply.substring(0, 150);
    }
    
    return reply;
  } catch (err) {
    console.error('AI生成失败，使用规则生成器:', err);
    return generateSmartReply(mentor, mood, content);
  }
}
```
