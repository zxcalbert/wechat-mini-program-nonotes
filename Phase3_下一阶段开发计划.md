## 三、数据库设计

### 3.1 思想孵化器集合 (incubator_reports)

```javascript
{
  "_id": String,              // 文档ID（系统自动生成）
  "_openid": String,          // 用户openid
  "content": String,          // 用户输入的初始想法
  "mentors": Array,           // 参与导师列表（默认：查理·芒格、张小龙、荣格）
  "dimensions": Array,        // 孵化维度（想法内核、目标对象、关键假设、阻力与风险、最小验证路径）
  "report": String,           // 完整孵化报告（Markdown格式）
  "status": String,           // 状态："pending"/"completed"/"failed"
  "createTime": Date,         // 创建时间
  "updateTime": Date          // 更新时间
}
```

### 3.2 结构分析集合 (structure_analysis_reports)

```javascript
{
  "_id": String,              // 文档ID（系统自动生成）
  "_openid": String,          // 用户openid
  "analysisType": String,     // 分析类型："company"/"product"
  "content": String,          // 用户输入的分析内容
  "dimensions": Array,        // 分析维度
  "report": String,           // 完整分析报告（Markdown格式）
  "status": String,           // 状态："pending"/"completed"/"failed"
  "createTime": Date,         // 创建时间
  "updateTime": Date          // 更新时间
}
```

### 3.3 数据库权限设置
- 权限规则：仅创建者可读写
- 索引：`_openid` + `createTime`（降序）

### 5.3 混合排序规则

```javascript
// 按createTime降序排序，所有类型数据混合展示
displayItems = [
  ...letters,           // 导师回信
  ...roundtables,       // 圆桌会议
  ...incubators,        // 思想孵化器
  ...structureAnalysis  // 结构分析
].sort((a, b) => b.createTime - a.createTime);
```
