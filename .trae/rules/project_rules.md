# 项目开发规范

## 核心原则

**保护现有原型，渐进式升级，禁止一次性重构**

---

## 一、重构禁令

### 禁止事项
- ❌ **禁止** 一次性大规模重构
- ❌ **禁止** 迁移到 TypeScript
- ❌ **禁止** 更换状态管理方案
- ❌ **禁止** 同时修改多个模块
- ❌ **禁止** 重构时添加新功能

### 单次变更限制
| 限制项 | 上限 |
|--------|------|
| 修改文件数 | ≤5个 |
| 代码行数 | ≤200行 |
| 影响模块 | ≤1个 |

---

## 二、渐进式开发流程

### 阶段一：文档化（当前）
- 记录现状
- 建立规范
- 修复合规性问题

### 阶段二：新功能试点（1-2月）
- 新功能遵循规范
- 选择1-2模块试点
- 积累经验

### 阶段三：核心重构（3-6月）
- 重构核心页面
- 充分测试验证
- 保持回滚能力

### 阶段四：全面生产化（6月+）
- 全面应用规范
- 性能优化完成
- 合规审核通过

---

## 三、安全防护措施

### 重构前
- **Git备份**：`git checkout -b backup/功能名-日期`
- **数据导出**：云开发控制台导出数据库
- **基线测试**：验证核心功能正常

### 重构中
- 每30分钟提交一次
- 每完成小功能点验证
- 添加关键位置日志

### 重构后
- 本地验证：编译无错误
- 真机验证：iOS+Android
- 云端验证：云函数正常

---

## 四、回滚策略

### 代码回滚
```bash
git revert HEAD                    # 回滚上一提交
git checkout backup/xxx            # 从备份恢复
```

### 云函数回滚
1. 登录云开发控制台
2. 云函数 → 版本历史
3. 选择稳定版本回滚

### 数据库回滚
1. 导出当前数据
2. 导入备份数据
3. 验证数据完整性

---

## 五、代码规范

### 风格
- 缩进：2空格
- 分号：必须
- 引号：单引号

### 异常处理
```javascript
try {
  await operation();
} catch (err) {
  console.error('[操作失败]', err);
  wx.showToast({ title: '操作失败', icon: 'none' });
}
```

### 边界处理
```javascript
// 安全获取
const value = obj?.property ?? defaultValue;

// 数组安全
if (!Array.isArray(arr)) return [];
```

## 六、Git工作流

```
main (生产)
  └── develop (开发)
        ├── feature/xxx
        ├── fix/xxx
        └── docs/xxx
```

### 提交格式
```
<type>(<scope>): <subject>

type: feat/fix/docs/style/refactor/test/chore
```

---

## 七、测试标准

### 必测项
- [ ] 核心功能正常
- [ ] 数据隔离正确
- [ ] 边界情况处理
- [ ] 错误提示清晰

### 回归测试
- 登录模块变更 → 测试新老用户流程
- 核心算法变更 → 单元测试+功能测试

---

## 八、安全红线

### 数据安全
- 敏感数据加密存储
- 查询强制用户ID过滤
- 定期自动备份

### 访问控制
- 登录状态检查
- 功能权限验证
- 请求频率限制

### 输入验证
- 长度/类型验证
- XSS防护
- 文件上传限制

---

## 九、合规要求

### 立即修复
- 头像昵称：使用`chooseAvatar`+`nickname`
- 隐私授权：实现`onNeedPrivacyAuthorization`
- 内容安全：接入`msgSecCheck`

---

## 十、决策流程

### 技术决策
1. 评估风险等级
2. 制定回滚方案
3. 小范围试点
4. 验证后推广

### 紧急处理
1. 立即回滚到稳定版本
2. 定位问题原因
3. 修复后重新验证
4. 记录经验教训
