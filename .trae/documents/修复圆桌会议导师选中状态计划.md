# 修复圆桌会议导师选中状态计划

## 问题描述
用户反馈在圆桌会议页面中，导师卡片被选中后没有显示预期的棕色背景（#8b4513）效果。

## 当前状态分析

### 1. 文件结构
- **页面文件**: `miniprogram/pages/roundtable/`
  - `roundtable.js` - 页面逻辑
  - `roundtable.wxml` - 页面结构
  - `roundtable.wxss` - 页面样式
  - `roundtable.json` - 页面配置

### 2. 问题根源分析
通过代码分析发现以下问题：

#### 2.1 样式冲突
在 [roundtable.wxss](file:///Users/bill/编程/invest-diary/miniprogram/pages/roundtable/roundtable.wxss) 中存在**两处** `.mentor-item.selected` 样式定义：

**第一处（第117-120行）**：
```css
.mentor-item.selected {
  background-color: var(--highlightColor, #8b4513);
  border-color: var(--highlightColor, #8b4513);
}
```

**第二处（第242-246行）**：
```css
.mentor-item.selected {
  background-color: #8b4513;
  border: 2px solid #8b4513;
  box-shadow: 0 2px 8px rgba(139, 69, 19, 0.3);
}
```

**问题**：两处样式定义可能导致CSS优先级冲突，影响最终渲染效果。

#### 2.2 功能开关状态
当前 `featureEnabled` 已设置为 `true`，但可能存在以下问题：
- 遮罩层逻辑可能影响页面渲染
- 样式加载顺序可能影响最终效果

#### 2.3 导师选中逻辑
在 [roundtable.wxml](file:///Users/bill/编程/invest-diary/miniprogram/pages/roundtable/roundtable.wxml#L28) 中，导师选中状态通过条件类名实现：
```xml
class="mentor-item {{selectedMentors.indexOf(mentor) > -1 ? 'selected' : ''}}"
```

逻辑正确，但样式可能未正确应用。
