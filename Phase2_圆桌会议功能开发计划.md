## 🔧 核心技术方案
### 1. 云函数扩展方案
```javascript
// replyToLetter云函数新增参数支持
exports.main = async (event, context) => {
  const { type = 'single', mentors = [], content } = event;
  
  if (type === 'single') {
    // 原有单导师逻辑，完全不变
    return singleMentorReply(event);
  } else if (type === 'roundtable') {
    // 新增多导师逻辑，独立实现
    return roundtableReply(event);
  }
}
```
**优点**：完全兼容现有接口，不影响旧版本调用

### 2. 上下文摘要方案
每轮回复后提取关键信息：`{mentorName, coreViewpoints, agreePoints, disagreePoints}`，总长度控制在500字以内，传递给下一位导师

### 3. 质量评分机制
从以下维度自动评分（满分1分）：
- 人设匹配度（0.4）：是否符合导师核心原则
- 内容相关性（0.3）：是否回应用户问题
- 观点独特性（0.2）：与之前导师回复是否有差异化
- 内容深度（0.1）：是否有启发性
- 低于0.7分自动触发重生成，最多重试2次
